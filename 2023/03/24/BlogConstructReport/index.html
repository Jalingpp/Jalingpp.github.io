Blog搭建报告

博客主题及其选取原因

这是我第一次搭建属于自己的网站。在此之前，曾努力维护过简书平台的博客账号，发表过几篇博文。但事实上，借助第三方平台维护的博客，在数据上是受制于人的，一旦哪天对方服务器崩掉了，我将失去所有我的数据。这一点对于研究区块链及去中心化存储的我来说，再熟悉不过了。借此机会，希望能搭建一个真正自己拥有的个人学术网站，如果能扩大一点点自己的影响力，那再好不过了。在经过几番挑选后，我选定在此主题下创建。

博客页面布局及其设计思路

本博客共包括主页、基本信息、发表成果、参与项目、博客分享五个部分。主页中较为醒目地展示我的名字、GitHub账号和所属单位等信息，并将以列表的形式展示我所有的博文。基本信息、发表成果、参与项目三个部分以静态页面的形式展示了我相关信息。博客分享页面将以时间顺序列出我发布的所有博文，点击标题将自动关联到详细页面。

博客功能实现及其技术选择

我将该博客定位为对我个人学术研究的总结与展示。选择的技术为Node.js+Hero+GitHub。

​ 第一步：安装Node.js

​ Node.js是一个用于创建服务器端应用程序的运行系统，它可以轻松构建网络或其他事件驱动的应用程序服务器。点击Node.js进入Node.js官网下载相应版本即可，安装时选项全部默认，一路点击Next。

​ 打开命令提示符，输入node -v和npm -v测试是否安装成功，若出现版本号则成功。

​ 第二步：添加国内镜像

​ 我们使用阿里的国内镜像进行加速，按Win+R打开命令提示符输入：

​ npm config set registry https://registry.npm.taobao.org

​ 第三步：安装git

​ Git是一个开源的分布式版本控制系统，可以有效、高速地处理从很小到非常大的项目版本管理，帮助我们把本地网页上传到Github。点击Git进入Git官网下载相应版本，默认安装即可。

​ 安装完成后在命令提示符中输入git –version验证是否安装成功。

​ 第四步：注册Github账号

​ Github作为全球最大的开源社区，相信朋友们都已经有了吧，如果没有，点击Github进入Github官网点击 Sign Up 注册账户。

​ 第五步：创建git仓库

​ 登录Github创建一个仓库。仓库项目名为”GitHub账户名+.github.io“。

​ 第六步：安装Hero

​ Hexo是一款基于Node.js的静态博客框架，依赖少易于安装使用，可以方便的生成静态网页托管在GitHub和Heroku，是搭建博客的首选框架。选择一个磁盘，新建一个文件夹用来存放博客文件。在该文件夹下右键单击鼠标，点击Git Bash Here，输入以下npm install hexo-cli -g命令即可安装。输入hexo -v即可检验是否安装成功。

​ 第七步：配置本地Hero

​ 在刚才新建的文件夹下再新建一个Hexo文件夹，在Hexo文件夹下右键单击鼠标，点击 Git Bash Here，依次输入以下hexo init命令和npm install命令即可初始化。接着输入hexo g生成静态网页，然后输入hexo s打开本地服务器，然后浏览器打开http://localhost:4000/，就可以预览我们的博客啦。显示INFO Hexo is running at http://0.0.0.0:4000/. Press Ctrl+C to stop.说明操作正确。按ctrl+c关闭本地服务器。

​ 第八步：连接GitHub与本地

​ （1）生成密钥：右键单击鼠标，点击 Git Bash Here输入命令git config –global user.name “Name”和git config –global user.email “Email”，其中Name和Email分别是注册GitHub时的用户名和邮箱。使用命令ssh-keygen -t rsa -C “Email”生成密钥。接下来一路点击回车，直到密钥被放在放在/c/Users/you/.ssh/id_rsa.pub.文件中。输入命令 clip < ~/.ssh/id_rsa.pub将密钥复制到粘贴板中。

​ （2）连接Github：在Github头像下面点击Settings，再点击SSH and GPG keys，新建一个SSH，名字任意。然后将刚才复制的密钥添加进去。

​ （3）本地连接Github：右键单击鼠标，点击Git Bash Here输入命令ssh -T git@github.com，如果显示Hi xxx! You’ve successfully authenticated, but GitHub does not provide shell access. 那就成功了。

​ 第九步：发布博客

​ 首先在博客根目录下右键打开Git Bash Here输入npm i hexo-deployer-git命令，安装一个扩展。然后输入hexo new post “article title”命令，新建一篇文章。然后打开C:\XXX\Hexo\source_posts的目录，可以发现下面多了一个.md文件，这个就是你的文章文件啦。编写完markdown文件后，根目录下右键打开Git Bash Here输入hexo g生成静态网页，然后输入hexo s可以本地预览效果，最后输入hexo d上传到github上。

​ 第十步：更换主题

​ 点击进入Hexo主题专栏可以看到很多Hexo主题。选定一个主题，点击进入其GitHub仓库。复制该项目地址后，打开 Hexo 文件夹下的 themes 目录，右键 Git Bash Here，输入git clone 主题地址 命令。下载完成后即可在 themes 目录下生成XXXX 文件夹，然后打开 Hexo 文件夹下的配置文件_config.yml ，找到关键字theme，修改参数为：theme：XXXX ，再次注意冒号后面有一个空格。博客根目录下右键 Git Bash Here，输入hexo clean，hexo g，hexo d命令予以更新。

博客样式设计及其美学考量

事实上，我想采用一个更简约的样式来作为个人的学术主页，但是并没有找到合适的可应用的主题，因此便选择了一个看起来相对个性化一点的主题。其整体上以深色人物照为底色，配色上较为大气。以下拉的方式进入正文，看起来酷炫一些。

博客制作过程中遇到的问题及其解决方法

问题一：_config.yml文件配置错误引发的无法将本地文件传输至github仓库中。

解决方案：找同学帮忙严格检查并发现错误后予以改正。

问题二：Hero上传本地项目至github时只能上传至master分支，而个人博客主页从main分支处访问，从而导致无法查看网页内容。

解决方案：从GitHub个人账号下的settings中修改默认分支为master并将博客设置为从master显示。

问题三：个人博客导航栏点击后出现的界面无法展示。

解决方案：仿照主页上post的展示方法给导航栏每个按钮添加链接，链接至一个markdown文件。

问题四：博文的详细页面正文之前有个宽宽的空白无法消除，影响美观。

解决方案：目前未找到解决方案，因为不熟悉前端代码。
